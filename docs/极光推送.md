## 通知行为

- **同会话覆盖**：同一频道/线程的多条推送，通知栏仅保留最新一条（类似微信）。
- **通知样式**：使用 Mattermost 风格（MessagingStyle + 发送者头像 + 快捷回复）。
- **Heads-up / 横幅**：自建渠道 `jpush_new_message_v5`（显示名「新消息通知」）+ `IMPORTANCE_HIGH`；JPush 本地通知勿用 `CATEGORY_MESSAGE`（华为/鸿蒙会归入「社交通讯」且无横幅）。
- **后台通知**：Android 通过 `setKeepLongConnInBackground(true)` 保活 JPush 长连接，确保 App 退后台/杀进程仍能即时收到推送。
- **仅 JPush 推送**：不使用 Mattermost 原有的 FCM 推送通路。

## 服务端推送 API extras 字段约定

JPush 推送 API 的 `android.extras` 需包含以下字段：

| 字段 | 必填 | 类型 | 说明 |
|------|------|------|------|
| `channel_id` | ✅ | string | 频道 ID（会话标识） |
| `post_id` | ✅ | string | 消息 post ID |
| `server_url` | ✅ | string | Mattermost 服务地址 |
| `sender_name` | ✅ | string | 发送者显示名 |
| `sender_id` | ✅ | string | 发送者用户 ID |
| `message` | ✅ | string | 消息正文（纯文本） |
| `root_id` | 否 | string | CRT 线程根帖 ID |
| `is_crt_enabled` | 否 | string | CRT 是否启用（`"true"` / `"false"`） |
| `channel_name` | 否 | string | 频道名（群聊通知展示用） |
| `category` | 否 | string | `"CAN_REPLY"` 启用快捷回复 |
| `team_id` | 否 | string | 团队 ID |
| `type` | 否 | string | 通知类型：`"message"` |
| `sub_type` | 否 | string | 通知子类型 |
| `override_icon_url` | 否 | string | 自定义头像 URL |
| `override_username` | 否 | string | 自定义用户名（Bot 使用） |
| `server_id` | 否 | string | 服务端 ID |
| `version` | 否 | string | 数据版本 |

### 极光 API options（兜底覆盖）

```json
{
  "options": {
    "override_msg_id": "conv_${channel_id}"
  }
}
```

- `override_msg_id`：极光侧同会话消息覆盖（1 天内有效），客户端 `notificationId` 覆盖策略不依赖此字段。

### 鸿蒙 4.x / 华为（离线走 HMS 时必配）

华为将 IM 类推送默认归入 **「社交通讯」**（`IMPORTANCE_DEFAULT`），系统设置里 **没有横幅开关**。客户端已创建 HIGH 渠道 `jpush_new_message_v5`，**服务端推送须带上 `channel_id`**，否则离线仍走 HMS 默认分类：

```json
{
  "notification": {
    "android": {
      "alert": "消息内容",
      "title": "标题",
      "channel_id": "jpush_new_message_v5"
    }
  },
  "options": {
    "third_party_channel": {
      "huawei": {
        "importance": "NORMAL",
        "category": "IM"
      }
    }
  }
}
```

说明见 [极光厂商消息分类](https://docs.jiguang.cn/jpush/client/Android/android_channel_id)。`importance: NORMAL` 表示服务与通讯类；横幅能否默认开启仍依赖客户端 HIGH 渠道 + 用户未手动关闭。

## 架构说明

### Android 原生层

- `JPushCustomReceiver`（自定义，`com.optibot.cn`）：拦截 JPush SDK 默认通知展示，使用 `CustomPushNotificationHelper.createNotificationBuilder()` 构建 Mattermost 风格通知（MessagingStyle + 头像 + 快捷回复）。
- `NotificationHelper.getNotificationId()`：CRT 模式按 `root_id.hashCode()`，否则按 `channel_id.hashCode()` 生成稳定 ID，同会话通知覆盖。
- `CustomPushNotificationHelper.createNotificationBuilder()`：通知构建统一入口，已适配新的 `notificationId` 策略。
- AndroidManifest.xml：`JPushCustomReceiver` 注册在 `JPushModuleReceiver` 之前。

### JS 层

- `app/init/jpush.ts`：
  - `setBackgroundEnable(true)` 开启后台长连接
  - `addNotificationListener` 分类型处理 `notificationOpened`（点击跳转）和 `notificationArrived`（前台 overlay）
- `app/init/app.ts`：仅调用 `JPushManager.init()`，不再调用 `PushNotifications.init()`（FCM 通路）

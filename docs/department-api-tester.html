部门管理 API 测试工具

基于 docs/department_api.md（含第 9 节用户搜索）、docs/team_version_api.md — Mattermost /api/v4

认证：Authorization: Bearer（Personal Access Token 或等价令牌）。与通讯录测试页的 X-API-KEY 不同。

下方「路径参数」输入框会随默认 ID 一键填充；修改默认项并失焦后会保存到本地。

路径参数（当前请求上下文）
部门管理
GET /api/v4/teams/{team_id}/departments

部门列表；查询参数 parent_id、page、per_page（parent_id 不传则不指定， -1: 根部门, >= 0: 指定 parent id）

POST /api/v4/teams/{team_id}/departments

创建部门

GET /api/v4/teams/{team_id}/departments/{department_id}

获取单个部门

PUT /api/v4/teams/{team_id}/departments/{department_id}

更新部门

DELETE /api/v4/teams/{team_id}/departments/{department_id}

软删除部门

团队版本与部门统计
GET /api/v4/teams/{team_id}/version

团队结构版本（缓存同步用，见 team_version_api）

GET /api/v4/teams/{team_id}/departments/stats

部门统计；可选查询参数 department_id：不传为全团队，传则按该部门（以服务端为准）

部门树形结构
GET /api/v4/teams/{team_id}/departments/tree

部门树（可选 parent_id、page、per_page）

GET /api/v4/teams/{team_id}/departments/{department_id}/children

直接子部门

GET /api/v4/teams/{team_id}/departments/{department_id}/ancestors

祖先部门链

部门成员管理
GET /api/v4/teams/{team_id}/departments/{department_id}/members

成员列表

GET /api/v4/teams/{team_id}/departments/without-members

未加入任何部门的成员列表（使用与成员列表相同的 page/per_page）

POST /api/v4/teams/{team_id}/departments/{department_id}/members

添加单个成员（JSON：user_id）

POST .../members/batch

批量添加成员

DELETE .../members/batch

批量移除成员（请求体 JSON）

DELETE /api/v4/teams/{team_id}/departments/{department_id}/members/{user_id}

移除单个成员（使用「成员操作目标 user_id」）

POST /api/v4/teams/{team_id}/departments/{department_id}/members/move

将成员从当前「路径中的 department_id」移到目标部门（body：user_id、target_department_id）

POST /api/v4/teams/{team_id}/departments/members/move-batch

批量移动（请求体 JSON）

用户部门查询
GET /api/v4/users/{user_id}/departments

查询参数 team_id 必填

用户搜索（Mattermost SearchUsers）
POST /api/v4/users/search

官方 SearchUsers；请求体 JSON：term 必填，可选 team_id、in_channel_id、allow_inactive、limit 等（见 docs/department_api.md §9）

员工联系人管理
GET /api/v4/users/{user_id}/contacts

可选 contact_type、page、per_page、granularity（不传：不返回 contact 对象；1：完整 contact；2：简洁 contact）

POST /api/v4/users/{user_id}/contacts

添加员工联系人

PUT /api/v4/users/{user_id}/contacts

更新员工联系人

DELETE /api/v4/users/{user_id}/contacts

删除关系（请求体 JSON）

GET /api/v4/users/{user_id}/contacts/version

联系人数据版本（缓存同步）

PUT /api/v4/users/{user_id}/contacts/version

更新联系人版本（变更联系人数据后可调用）

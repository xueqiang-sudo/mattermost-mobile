## Overview

### What is Team Version?

Team Version 是一个字符串标识符，用于跟踪团队结构和成员关系的变化。每当团队的以下数据发生变化时，版本号会自动更新：

- ✅ 团队成员添加或移除
- ✅ 部门（Department）创建、更新或删除
- ✅ 部门成员添加或移除
- ✅ 部门成员在不同部门间移动

### Why Use Team Version?

**Problem:** 
在大型组织中，团队结构可能频繁变化。客户端如果一直使用缓存的旧数据，会导致：
- 显示过期的部门结构
- 遗漏新加入的成员
- 显示已删除的部门
- 数据不一致的用户体验

**Solution:**
通过版本管理机制，客户端可以：
- 轻量级地检测数据是否过期（只需检查版本号）
- 仅在必要时才重新获取完整数据
- 减少不必要的网络请求和服务器负载
- 提供实时的数据同步体验

## Get Team Version API

### Endpoint

```
GET /api/v4/teams/{team_id}/version
```

### Authentication

**Required:** Yes

All requests must include a valid authentication token in the Authorization header:

```http
Authorization: Bearer YOUR_ACCESS_TOKEN
```

### Permissions

**Required Permission:** `PermissionViewTeam`

The user must have permission to view the team. This is typically granted to:
- Team members
- Team admins
- System admins

### Path Parameters

| Parameter | Type   | Required | Description                | Example                    |
|-----------|--------|----------|----------------------------|----------------------------|
| team_id   | string | Yes      | The unique ID of the team  | `abc123def456ghi789jklmno` |

**Note:** Team IDs are 26-character alphanumeric strings generated by Mattermost.

### Query Parameters

This endpoint does not accept any query parameters.

### Request Example

```bash
curl -X GET \
  'http://localhost:8065/api/v4/teams/abc123def456ghi789jklmno/version' \
  -H 'Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...'
```

### Success Response

**Status Code:** `200 OK`

**Response Body:**
```json
{
  "team_id": "abc123def456ghi789jklmno",
  "version": "v1713340800000",
  "updated_at": 1713340800000
}
```

**Response Fields:**

| Field      | Type   | Description                                              | Example              |
|------------|--------|----------------------------------------------------------|----------------------|
| team_id    | string | The ID of the team                                       | `abc123def456...`    |
| version    | string | Current version string (format: `v{timestamp}`)          | `v1713340800000`     |
| updated_at | number | Unix timestamp in milliseconds when response was generated | `1713340800000`      |

**Version String Format:**
- Format: `"v{unix_timestamp_milliseconds}"`
- Example: `"v1713340800000"`
- The timestamp represents when the version was last updated
- Versions are monotonically increasing (newer versions have larger numbers)

### Error Responses

#### 400 Bad Request

**Cause:** Missing or invalid `team_id` parameter

**Example:**
```json
{
  "id": "api.context.invalid_param.app_error",
  "message": "Invalid or missing team_id parameter in request URL",
  "status_code": 400
}
```

#### 401 Unauthorized

**Cause:** Missing or invalid authentication token

**Example:**
```json
{
  "id": "api.context.session_expired.app_error",
  "message": "Invalid or expired session, please login again",
  "status_code": 401
}
```

#### 403 Forbidden

**Cause:** User does not have permission to view the team

**Example:**
```json
{
  "id": "api.context.permissions.app_error",
  "message": "You do not have the appropriate permissions",
  "status_code": 403,
  "permission": "view_team"
}
```

#### 404 Not Found

**Cause:** Team with the specified ID does not exist

**Example:**
```json
{
  "id": "store.sql_team.get.missing_app_error",
  "message": "We couldn't find the existing team",
  "status_code": 404
}
```

#### 500 Internal Server Error

**Cause:** Server error while retrieving version from database

**Example:**
```json
{
  "id": "api.team.get_version_error",
  "message": "Unable to retrieve team version",
  "status_code": 500,
  "details": "connection timeout"
}
```

## Automatic Version Updates

### When is the Version Updated?

The team version is **automatically updated** by the server when any of the following operations occur:

#### 1. Department Operations

| Operation                      | API Endpoint                                          | Method |
|--------------------------------|-------------------------------------------------------|--------|
| Create Department              | `/teams/{team_id}/departments`                        | POST   |
| Update Department              | `/teams/{team_id}/departments/{department_id}`        | PUT    |
| Delete Department              | `/teams/{team_id}/departments/{department_id}`        | DELETE |

#### 2. Department Member Operations

| Operation                      | API Endpoint                                                     | Method |
|--------------------------------|------------------------------------------------------------------|--------|
| Add Department Member          | `/teams/{team_id}/departments/{department_id}/members`           | POST   |
| Remove Department Member       | `/teams/{team_id}/departments/{department_id}/members/{user_id}` | DELETE |
| Batch Add Members              | `/teams/{team_id}/departments/{department_id}/members/batch`     | POST   |
| Batch Remove Members           | `/teams/{team_id}/departments/{department_id}/members/batch`     | DELETE |
| Move Department Member         | `/teams/{team_id}/departments/{department_id}/members/move`      | POST   |
| Batch Move Members             | `/teams/{team_id}/departments/members/move-batch`                | POST   |

### Update Mechanism

When any of the above operations succeed, the server automatically:

1. Generates a new version string using the current timestamp: `v{milliseconds}`
2. Updates the `team_versions` table in the database
3. Uses UPSERT logic to ensure concurrency safety:
   - **PostgreSQL:** `ON CONFLICT (team_id) DO UPDATE`
   - **MySQL:** `ON DUPLICATE KEY UPDATE`

**Important Notes:**
- ❌ There is NO manual API to update the team version
- ✅ Updates are handled automatically by the server
- ⚠️ Failed version updates are logged but don't fail the main operation
- 🔄 Multiple rapid changes will result in multiple version updates

### Implementation Details

Server-side implementation (for reference):

```go
func updateTeamVersion(c *Context, teamId string) {
    // Generate new version based on timestamp
    newVersion := fmt.Sprintf("v%d", model.GetMillis())
    
    // Try to update the version in database
    appErr := c.App.Srv().Store().Department().UpdateTeamVersion(teamId, newVersion)
    if appErr != nil {
        // Log error but don't fail the operation
        c.Logger.Warn("Failed to update team version", 
            mlog.String("team_id", teamId), 
            mlog.Err(appErr))
    }
}
```

## Cache Synchronization Strategy

### Recommended Approach: Polling with Version Check

This is the most reliable and widely-used strategy for keeping client caches in sync.

#### Step 1: Initial Data Load

When the application first loads or when a user navigates to a team:

```javascript
async function initializeTeamCache(teamId) {
  // 1. Fetch full department data
  const departmentsResponse = await fetch(
    `/api/v4/teams/${teamId}/departments`,
    { headers: { 'Authorization': `Bearer ${token}` } }
  );
  const departmentsData = await departmentsResponse.json();
  
  // 2. Fetch current version
  const versionResponse = await fetch(
    `/api/v4/teams/${teamId}/version`,
    { headers: { 'Authorization': `Bearer ${token}` } }
  );
  const versionData = await versionResponse.json();
  
  // 3. Store both in local storage
  localStorage.setItem(`team_${teamId}_departments`, JSON.stringify(departmentsData));
  localStorage.setItem(`team_${teamId}_version`, versionData.version);
  
  console.log(`Cache initialized with version ${versionData.version}`);
  
  return {
    departments: departmentsData,
    version: versionData.version
  };
}
```

#### Step 2: Periodic Version Check

Set up a polling mechanism to check for updates:

```javascript
class TeamVersionSync {
  constructor(teamId, token, options = {}) {
    this.teamId = teamId;
    this.token = token;
    this.pollInterval = options.pollInterval || 30000; // Default: 30 seconds
    this.onUpdate = options.onUpdate || null;
    this.onError = options.onError || null;
    this.isRunning = false;
    this.intervalId = null;
  }
  
  async checkVersion() {
    try {
      const response = await fetch(
        `/api/v4/teams/${this.teamId}/version`,
        { headers: { 'Authorization': `Bearer ${this.token}` } }
      );
      
      if (!response.ok) {
        throw new Error(`HTTP ${response.status}: ${response.statusText}`);
      }
      
      const data = await response.json();
      const storedVersion = localStorage.getItem(`team_${this.teamId}_version`);
      
      // Compare versions
      if (data.version !== storedVersion) {
        console.log(`Version changed: ${storedVersion} → ${data.version}`);
        
        // Notify listeners
        if (this.onUpdate) {
          await this.onUpdate(data.version, storedVersion);
        }
        
        // Update stored version
        localStorage.setItem(`team_${this.teamId}_version`, data.version);
      }
      
      return data.version;
    } catch (error) {
      console.error('Version check failed:', error);
      if (this.onError) {
        this.onError(error);
      }
      throw error;
    }
  }
  
  start() {
    if (this.isRunning) {
      console.warn('Version sync is already running');
      return;
    }
    
    this.isRunning = true;
    console.log(`Starting version sync for team ${this.teamId} (interval: ${this.pollInterval}ms)`);
    
    // Check immediately
    this.checkVersion().catch(console.error);
    
    // Then poll at intervals
    this.intervalId = setInterval(() => {
      this.checkVersion().catch(console.error);
    }, this.pollInterval);
  }
  
  stop() {
    if (!this.isRunning) {
      return;
    }
    
    this.isRunning = false;
    clearInterval(this.intervalId);
    this.intervalId = null;
    console.log(`Stopped version sync for team ${this.teamId}`);
  }
}
```

## Implementation Examples

### React Hook Implementation

```typescript
import { useState, useEffect, useCallback } from 'react';

interface TeamVersion {
  team_id: string;
  version: string;
  updated_at: number;
}

interface UseTeamVersionOptions {
  pollInterval?: number;
  enabled?: boolean;
  onVersionChange?: (newVersion: string, oldVersion: string) => void;
}

export function useTeamVersion(
  teamId: string,
  token: string,
  options: UseTeamVersionOptions = {}
) {
  const {
    pollInterval = 30000,
    enabled = true,
    onVersionChange
  } = options;
  
  const [version, setVersion] = useState<string | null>(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState<Error | null>(null);
  const [lastChecked, setLastChecked] = useState<Date | null>(null);
  
  const fetchVersion = useCallback(async () => {
    if (!teamId || !token) {
      return;
    }
    
    try {
      setLoading(true);
      setError(null);
      
      const response = await fetch(
        `/api/v4/teams/${teamId}/version`,
        { headers: { 'Authorization': `Bearer ${token}` } }
      );
      
      if (!response.ok) {
        throw new Error(`Failed to fetch version: ${response.statusText}`);
      }
      
      const data: TeamVersion = await response.json();
      const oldVersion = version;
      
      setVersion(data.version);
      setLastChecked(new Date());
      
      // Notify if version changed
      if (oldVersion && data.version !== oldVersion && onVersionChange) {
        onVersionChange(data.version, oldVersion);
      }
      
    } catch (err) {
      setError(err instanceof Error ? err : new Error(String(err)));
      console.error('Failed to fetch team version:', err);
    } finally {
      setLoading(false);
    }
  }, [teamId, token, version, onVersionChange]);
  
  useEffect(() => {
    if (!enabled) {
      return;
    }
    
    // Initial fetch
    fetchVersion();
    
    // Set up polling
    const intervalId = setInterval(fetchVersion, pollInterval);
    
    // Cleanup
    return () => clearInterval(intervalId);
  }, [enabled, pollInterval, fetchVersion]);
  
  return {
    version,
    loading,
    error,
    lastChecked,
    refetch: fetchVersion
  };
}

// Usage in component
function TeamDepartments({ teamId }: { teamId: string }) {
  const [departments, setDepartments] = useState([]);
  const token = useAuthToken();
  
  const { version, loading, error } = useTeamVersion(teamId, token, {
    pollInterval: 30000,
    onVersionChange: async (newVersion) => {
      console.log('Version changed, refreshing departments...');
      await refreshDepartments();
    }
  });
  
  const refreshDepartments = async () => {
    const response = await fetch(`/api/v4/teams/${teamId}/departments`, {
      headers: { 'Authorization': `Bearer ${token}` }
    });
    const data = await response.json();
    setDepartments(data.departments);
  };
  
  useEffect(() => {
    refreshDepartments();
  }, [teamId]);
  
  if (loading) return <div>Loading...</div>;
  if (error) return <div>Error: {error.message}</div>;
  
  return (
    <div>
      <p>Current version: {version}</p>
      {/* Render departments */}
    </div>
  );
}
```

### Angular Service

```typescript
import { Injectable, OnDestroy } from '@angular/core';
import { HttpClient, HttpHeaders } from '@angular/common/http';
import { BehaviorSubject, Observable, interval, Subscription } from 'rxjs';
import { switchMap, startWith, distinctUntilChanged } from 'rxjs/operators';

interface TeamVersion {
  team_id: string;
  version: string;
  updated_at: number;
}

@Injectable({
  providedIn: 'root'
})
export class TeamVersionService implements OnDestroy {
  private versionSubject = new BehaviorSubject<string | null>(null);
  public version$: Observable<string | null> = this.versionSubject.asObservable();
  
  private pollingSubscription: Subscription | null = null;
  
  constructor(private http: HttpClient) {}
  
  startPolling(teamId: string, token: string, intervalMs: number = 30000): void {
    this.stopPolling();
    
    const headers = new HttpHeaders({
      'Authorization': `Bearer ${token}`
    });
    
    this.pollingSubscription = interval(intervalMs)
      .pipe(
        startWith(0),
        switchMap(() => 
          this.http.get<TeamVersion>(`/api/v4/teams/${teamId}/version`, { headers })
        ),
        distinctUntilChanged((a, b) => a.version === b.version)
      )
      .subscribe({
        next: (data) => {
          this.versionSubject.next(data.version);
          console.log(`Team version updated: ${data.version}`);
        },
        error: (err) => {
          console.error('Failed to fetch team version:', err);
        }
      });
  }
  
  stopPolling(): void {
    if (this.pollingSubscription) {
      this.pollingSubscription.unsubscribe();
      this.pollingSubscription = null;
    }
  }
  
  ngOnDestroy(): void {
    this.stopPolling();
  }
}

// Usage in component
@Component({

  template: `

  }
  
  ngOnInit(): void {
    const teamId = 'abc123...';
    const token = this.authService.getToken();
```

### Plain JavaScript (Vanilla)

```javascript
class TeamCacheManager {
  constructor(teamId, authToken) {
    this.teamId = teamId;
    this.authToken = authToken;
    this.currentVersion = null;
    this.cacheKey = `team_${teamId}_`;
    this.pollInterval = null;
    this.listeners = [];
  }
  
  async init() {
    // Load from cache if available
    const cachedVersion = localStorage.getItem(this.cacheKey + 'version');
    const cachedData = localStorage.getItem(this.cacheKey + 'departments');
    
    if (cachedVersion && cachedData) {
      this.currentVersion = cachedVersion;
      console.log('Loaded from cache:', cachedVersion);
      return JSON.parse(cachedData);
    }
    
    // Otherwise fetch fresh data
    return await this.refresh();
  }
  
  async getVersion() {
    const response = await fetch(`/api/v4/teams/${this.teamId}/version`, {
      headers: {
        'Authorization': `Bearer ${this.authToken}`
      }
    });
    
    if (!response.ok) {
      throw new Error(`HTTP ${response.status}`);
    }
    
    return await response.json();
  }
  
  async refresh() {
    console.log('Refreshing team data...');
    
    try {
      // Fetch version
      const versionData = await this.getVersion();
      
      // Fetch departments
      const deptsResponse = await fetch(
        `/api/v4/teams/${this.teamId}/departments`,
        {
          headers: {
            'Authorization': `Bearer ${this.authToken}`
          }
        }
      );
      
      if (!deptsResponse.ok) {
        throw new Error('Failed to fetch departments');
      }
      
      const departmentsData = await deptsResponse.json();
      
      // Update cache
      this.currentVersion = versionData.version;
      localStorage.setItem(this.cacheKey + 'version', versionData.version);
      localStorage.setItem(this.cacheKey + 'departments', JSON.stringify(departmentsData));
      
      // Notify listeners
      this.notifyListeners(versionData.version);
      
      return departmentsData;
      
    } catch (error) {
      console.error('Refresh failed:', error);
      throw error;
    }
  }
  
  async checkForUpdates() {
    if (!this.currentVersion) {
      return false;
    }
    
    try {
      const versionData = await this.getVersion();
      
      if (versionData.version !== this.currentVersion) {
        console.log(`Version changed: ${this.currentVersion} → ${versionData.version}`);
        await this.refresh();
        return true;
      }
      
      return false;
    } catch (error) {
      console.error('Version check failed:', error);
      return false;
    }
  }
  
  startAutoSync(intervalMs = 30000) {
    this.stopAutoSync();
    
    console.log(`Starting auto-sync (interval: ${intervalMs}ms)`);
    
    // Check immediately
    this.checkForUpdates().catch(console.error);
    
    // Then poll
    this.pollInterval = setInterval(() => {
      this.checkForUpdates().catch(console.error);
    }, intervalMs);
  }
  
  stopAutoSync() {
    if (this.pollInterval) {
      clearInterval(this.pollInterval);
      this.pollInterval = null;
      console.log('Auto-sync stopped');
    }
  }
  
  onVersionChange(callback) {
    this.listeners.push(callback);
    return () => {
      this.listeners = this.listeners.filter(cb => cb !== callback);
    };
  }
  
  notifyListeners(newVersion) {
    this.listeners.forEach(callback => {
      try {
        callback(newVersion, this.currentVersion);
      } catch (error) {
        console.error('Listener error:', error);
      }
    });
  }
  
  clearCache() {
    localStorage.removeItem(this.cacheKey + 'version');
    localStorage.removeItem(this.cacheKey + 'departments');
    this.currentVersion = null;
    console.log('Cache cleared');
  }
}

// Usage
const cacheManager = new TeamCacheManager('abc123...', 'your_token_here');

// Initialize
cacheManager.init().then(departments => {
  console.log('Initial data loaded:', departments);
  
  // Start auto-sync
  cacheManager.startAutoSync(30000);
  
  // Listen for updates
  cacheManager.onVersionChange((newVersion, oldVersion) => {
    console.log('Data updated!');
    // Update UI here
  });
```

## Best Practices

### 1. Polling Interval

**Recommendation:** 30-60 seconds for most applications

**Considerations:**
- Shorter intervals (10-15s): More responsive but higher server load
- Longer intervals (60-120s): Less server load but slower updates
- Adaptive intervals: Start with 10s, increase to 60s after 5 minutes of no changes

```javascript
// Adaptive polling example
let pollInterval = 10000; // Start with 10 seconds
let noChangeCount = 0;

setInterval(async () => {
  const hasChanged = await checkForUpdates();
  
  if (!hasChanged) {
    noChangeCount++;
    if (noChangeCount > 30 && pollInterval < 60000) {
      // After 5 minutes of no changes, reduce frequency
      pollInterval = 60000;
      restartPolling(pollInterval);
    }
  } else {
    noChangeCount = 0;
    pollInterval = 10000; // Reset to frequent checking
    restartPolling(pollInterval);
  }
}, pollInterval);
```

### 2. Error Handling

Always implement robust error handling:

```javascript
async function safeVersionCheck() {
  const maxRetries = 3;
  let retryCount = 0;
  
  while (retryCount < maxRetries) {
    try {
      return await checkVersion();
    } catch (error) {
      retryCount++;
      
      if (retryCount === maxRetries) {
        console.error('Max retries reached:', error);
        showOfflineMode();
        return null;
      }
      
      // Exponential backoff
      const delay = Math.pow(2, retryCount) * 1000;
      console.log(`Retry ${retryCount} in ${delay}ms...`);
      await sleep(delay);
    }
  }
}
```

## Troubleshooting

### Common Issues

#### Issue 1: Version Never Changes

**Symptoms:**
- Polling returns the same version indefinitely
- Departments don't update even after modifications

**Possible Causes:**
1. No actual changes to team structure
2. Database write failures (check server logs)
3. Caching layer issues (Redis, etc.)

**Solutions:**
```bash
# Check server logs for version update errors
tail -f /var/log/mattermost/mattermost.log | grep "team version"

# Verify database entry
psql -U mmuser -d mattermost -c "SELECT * FROM team_versions WHERE team_id='abc123...';"

# Manually trigger a department update to test
curl -X PUT /api/v4/teams/abc123/departments/1 \
  -H "Authorization: Bearer TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"name":"Test Update"}'
```

#### Issue 2: Too Many Version Updates

**Symptoms:**
- Version changes every few seconds
- Excessive network traffic

**Possible Causes:**
1. Automated scripts making frequent changes
2. Bug causing repeated updates
3. Multiple clients triggering updates

**Solutions:**
```javascript
// Implement debouncing on the client side
let updateTimeout = null;

function debouncedRefresh() {
  if (updateTimeout) {
    clearTimeout(updateTimeout);
  }
  
  updateTimeout = setTimeout(async () => {
    await refreshData();
  }, 2000); // Wait 2 seconds after last version change
}

// On server: Check for update storms
SELECT team_id, COUNT(*) as update_count
FROM team_versions_audit
WHERE updated_at > NOW() - INTERVAL '1 hour'
GROUP BY team_id
HAVING COUNT(*) > 100;
```

#### Issue 3: Version Check Fails with 401

**Symptoms:**
- Intermittent 401 errors during version checks
- Works after page refresh

**Possible Causes:**
1. Token expiration
2. Session invalidation
3. Clock skew between client and server

**Solutions:**
```javascript
async function checkVersionWithAuthRefresh() {
  try {
    return await fetchVersion();
  } catch (error) {
    if (error.status === 401) {
      console.log('Token expired, refreshing...');
      
      // Attempt to refresh token
      const newToken = await refreshToken();
      
      if (newToken) {
        // Retry with new token
        return await fetchVersion(newToken);
      } else {
        // Redirect to login
        window.location.href = '/login';
      }
    }
    throw error;
  }
}
```

#### Issue 4: Multiple Tabs Out of Sync

**Symptoms:**
- Different tabs show different data
- Updates in one tab don't reflect in others

**Solution: Cross-tab synchronization**
```javascript
// Listen for storage events (fires when other tabs modify localStorage)
window.addEventListener('storage', (event) => {
  if (event.key === `team_${teamId}_version`) {
    console.log('Another tab updated the version');
    
    // Reload data in this tab too
    refreshData();
  }
});

// Or use BroadcastChannel API for more control
const channel = new BroadcastChannel('team_updates');

channel.onmessage = (event) => {
  if (event.data.type === 'VERSION_UPDATE' && event.data.teamId === teamId) {
    console.log('Received update from another tab');
    refreshData();
  }
};

// Broadcast when updating
function broadcastUpdate(teamId, newVersion) {
  channel.postMessage({
    type: 'VERSION_UPDATE',
    teamId: teamId,
    version: newVersion
  });
}
```

## Performance Considerations

### Server Load

Each version check requires:
- 1 HTTP request
- 1 database query (SELECT)
- Authentication and authorization checks

**Estimated load:**
- 1000 clients polling every 30s = ~33 requests/second
- With proper caching and indexing, this is negligible

### Optimization Tips

1. **Database Indexing:**
   ```sql
   -- Ensure primary key index exists
   CREATE UNIQUE INDEX IF NOT EXISTS team_versions_pkey 
   ON team_versions(team_id);
   ```

2. **Application-Level Caching:**
   ```go
   // Server-side: Cache version in memory for 5 seconds
   var versionCache = make(map[string]*CachedVersion)
   
   type CachedVersion struct {
       Version   string
       ExpiresAt int64
   }
   
   func GetTeamVersionWithCache(teamId string) (string, error) {
       cached, ok := versionCache[teamId]
       if ok && time.Now().UnixMilli() < cached.ExpiresAt {
           return cached.Version, nil
       }
       
       // Fetch from database
       version, err := store.GetTeamVersion(teamId)
       if err != nil {
           return "", err
       }
       
       // Cache for 5 seconds
       versionCache[teamId] = &CachedVersion{
           Version:   version,
           ExpiresAt: time.Now().Add(5 * time.Second).UnixMilli(),
       }
       
       return version, nil
   }
   ```

3. **Client-Side Caching:**
   ```javascript
   // Don't check more than once per 10 seconds
   let lastCheckTime = 0;
   
   async function throttledVersionCheck() {
     const now = Date.now();
     if (now - lastCheckTime < 10000) {
       return cachedVersion;
     }
     
     const version = await fetchVersion();
     lastCheckTime = now;
     return version;
   }
   ```

## Migration Guide

### Upgrading from No Version Management

If you're adding version management to an existing application:

1. **Database Migration:**
   ```sql
   -- Create team_versions table
   CREATE TABLE IF NOT EXISTS team_versions (
       team_id VARCHAR(26) PRIMARY KEY,
       version VARCHAR(50) NOT NULL DEFAULT 'v0',
       updated_at BIGINT NOT NULL DEFAULT 0
   );
   
   -- Initialize existing teams with v0
   INSERT INTO team_versions (team_id, version, updated_at)
   SELECT Id, 'v0', EXTRACT(EPOCH FROM NOW()) * 1000
   FROM Teams
   ON CONFLICT (team_id) DO NOTHING;
   ```

2. **Client Updates:**
   ```javascript
   // Add version checking to existing code
   const OLD_WAY = {
     // Before: Just fetch data periodically
     fetchData: () => fetch('/api/departments').then(r => r.json())
   };
   
   const NEW_WAY = {
     // After: Check version first
     checkAndUpdate: async () => {
       const version = await fetchVersion();
       if (version !== cachedVersion) {
         await fetchData();
       }
     }
   };
   ```

3. **Gradual Rollout:**
   - Deploy server-side changes first
   - Enable version tracking in background
   - Gradually enable client-side version checking
   - Monitor for issues before full rollout

## FAQ

**Q: Can I manually update the team version?**  
A: No, the version is automatically managed by the server. Manual updates are not supported.

**Q: What happens if the version update fails?**  
A: The main operation (e.g., adding a member) still succeeds. The version update failure is logged but doesn't block the operation.

**Q: How long should I cache the version?**  
A: Don't cache the version for too long. Poll every 30-60 seconds to detect changes promptly.

**Q: Can I use this for real-time collaboration?**  
A: For true real-time needs, consider implementing WebSocket notifications instead of polling.

**Q: Does this work across multiple servers (cluster mode)?**  
A: Yes, the version is stored in the shared database, so it works correctly in clustered environments.

**Q: What's the maximum version string length?**  
A: The database allows up to 50 characters. The format `v{timestamp}` uses about 14-15 characters.
